# Airport Overlay — Design Spec

**Date:** 2026-06-07  
**Status:** Approved

## Overview

Add airport runway overlays to RadarView. Airports are drawn as filled rotated rectangles (one per runway) using real OurAirports geometry. Data is pre-processed at build time into a static JSON file, lazy-fetched at runtime, and filtered to the current radar radius. Users can configure which airport tiers to display. Hovering an airport shows its name and ICAO code.

---

## Scope

- **RadarView only** — MapView is out of scope
- **Rendering:** filled runway polygons (scaled rectangles, rotated to true bearing)
- **Interaction:** hover tooltip (name + ICAO). No click-to-pin.
- **Data:** OurAirports CSVs → `public/airports.json`, generated by a one-time script

---

## Data Pipeline

### Build-time script

`scripts/generate-airports.ts` — run via `npm run generate:airports`

1. Fetch from OurAirports CDN:
   - `https://davidmegginson.github.io/ourairports-data/airports.csv`
   - `https://davidmegginson.github.io/ourairports-data/runways.csv`
2. Filter airports: `type ∈ {large_airport, medium_airport, small_airport}`, valid `lat_deg`/`lon_deg`
3. Join runways by `airport_ident`; filter runways with valid `le_latitude_deg`, `le_longitude_deg`, `he_latitude_deg`, `he_longitude_deg`
4. Emit `public/airports.json` — array of `Airport` objects (see Data Model)

Script is not wired into dev/build automatically. Run manually when data needs refreshing. Output committed to the repo — no network dependency at dev or build time.

Estimated output size: ~2–3 MB uncompressed, ~500 KB gzipped (all three tiers globally).

## Data Model

```ts
// src/types/airport.ts

export type AirportType = 'large_airport' | 'medium_airport' | 'small_airport'

export interface RunwayEnd {
  ident: string   // e.g. "10L", "28R"
  lat: number
  lon: number
}

export interface Runway {
  le: RunwayEnd
  he: RunwayEnd
  widthFt: number
  lengthFt: number
}

export interface Airport {
  icao: string
  iata: string
  name: string
  lat: number
  lon: number
  type: AirportType
  runways: Runway[]
}
```

## Runtime Architecture

### `src/api/airports.ts`

Lazy-fetches `/airports.json` once on first call. Caches result in module scope. Returns `Airport[]`.

```ts
let cache: Airport[] | null = null

export async function fetchAirports(): Promise<Airport[]> {
  if (cache) return cache
  const res = await fetch('/airports.json')
  cache = await res.json()
  return cache
}
```

### `src/hooks/useAirports.ts`

Calls `fetchAirports()` on mount. Filters by:
- `settings.airportTypes` (tier whitelist)
- `haversineKm(settings center, airport) ≤ settings.radiusKm`

Returns `{ airports: Airport[], loading: boolean, error: string | null }`.

No new Zustand store — airports are static data, not reactive between updates.

### `RadarView.tsx`

- Calls `useAirports()`, passes `airports` to `RadarCanvas`
- Extends mousemove handler: calls `findClosestAirport(airports, mouseX, mouseY, scale)` after existing aircraft hit-test
- Aircraft take priority: only check airports if no aircraft is hovered
- Stores `hoveredAirport: Airport | null` in local state
- Renders hover tooltip using existing tooltip component/style: `{name} ({icao})`

### `RadarCanvas.ts`

New `drawAirports(ctx, airports, center, radiusKm, width, height)` function called in the draw loop before aircraft.

**Per runway:**
1. Project `le` and `he` lat/lon → canvas coords via `latLonToCanvas()`
2. Compute center point (midpoint of projected le/he)
3. Compute rendered length in px (distance between projected points)
4. Compute rendered width in px: `widthFt × 0.0003048 / radiusKm × scale`
5. Compute rotation angle: `Math.atan2(dy, dx)`
6. Skip draw if rendered length < 8px (too small to be legible)
7. `ctx.save()` → translate to center → rotate → `fillRect` → `strokeRect` → `ctx.restore()`
8. Draw ICAO label near airport center if rendered length ≥ 12px

**Styling:**
- Fill: `rgba(74, 222, 128, 0.12)`
- Stroke: `rgba(74, 222, 128, 0.45)`, line width 0.8px
- Label: `rgba(74, 222, 128, 0.65)`, 9px monospace

### `findClosestAirport()`

Utility in `src/lib/geoUtils.ts` (or inline in RadarView). Checks distance from mouse to each airport's canvas center point. Returns airport if distance ≤ 18px, else null.

## Error Handling

- Fetch failure: `useAirports` sets `error` state, RadarView renders nothing (airports are decorative — silent degradation is acceptable)
- Airports with zero valid runways after filtering: render 3px dot at airport center (same green fill, 0.7 opacity)
- Missing `iata`: fall back to `icao` for display

---

## Out of Scope

- MapView airport rendering
- Clicking airports to pin info cards
- Automatic data refresh (airports.json is static)
- Elevation / 3D data
